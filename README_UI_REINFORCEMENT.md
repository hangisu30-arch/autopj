# UI Reinforcement

이번 단계는 UI를 크게 바꾸지 않고, 생성기 신뢰성과 디버깅 편의성을 높이는 방향으로 보강한다.

## 추가된 것
- `분석/계획/검증 보기` 헤더와 `디버그 결과 새로고침` 버튼
- `.autopj_debug/*.json` 및 `apply_report.json` 자동 로드
- 요약 뷰
- 탭 추가
  - `분석 결과`
  - `생성 계획`
  - `검증/복구`
  - `적용 보고서`

## 표시되는 정보
- 분석 결과의 프로젝트/도메인/필드/기능 분류
- 백엔드 및 프론트 생성 예정 파일 목록
- 전역 검증 결과와 자동 복구 계획
- 실제 적용 보고서(written/failed)

## 핵심 파일
- `app/ui/debug_artifacts.py`
- `app/ui/main_window.py`

## 의도
- 생성 전후 상태를 UI에서 직접 확인 가능
- 실패 원인을 로그 외에도 구조화된 결과로 확인 가능
- 이후 `실패 파일만 재생성` UX 확장 기반 확보

수정 요약

문제 분석
- 업로드된 스크린샷은 달력 본문이 거의 비어 보이고, 상단 툴바만 어색하게 남아 있는 상태였습니다.
- 기존 생성 로직의 generic calendar JSP는 schedule-layout에서 sidepanel을 먼저 배치해 본문 폭이 비정상적으로 줄어들 수 있었습니다.
- 또한 달력 본문과 선택 일정 패널이 JavaScript 렌더링에 과도하게 의존해서, JS가 실패하거나 늦게 로드되면 화면이 비어 보일 수 있었습니다.
- 기존 normalize 로직은 legacy fullcalendar 화면은 교체했지만, 이미 AUTOPJ 형식으로 생성된 약한 calendar JSP는 다시 교체하지 못했습니다.

적용한 수정
- app/io/execution_core_apply.py
  - _build_entity_calendar_jsp() 개선
    - calendar-board를 schedule-sidepanel보다 먼저 배치
    - calendarCells 기반 서버 렌더링 fallback 추가
    - selectedDateSchedules 기반 우측 패널 서버 렌더링 fallback 추가
    - currentYear/currentMonth가 있으면 툴바 제목을 서버 렌더링
  - _normalize_calendar_jsp() 강화
    - server-rendered calendarCells 부재 시 재정규화
    - selectedDateSchedules 목록 부재 시 재정규화
    - sidepanel이 calendar-board보다 먼저 오는 약한 AUTOPJ calendar도 재정규화
- tests/test_schedule_entity_calendar_ssr_fallback.py 추가
  - generic entity calendar가 서버 렌더링 fallback과 올바른 레이아웃 순서를 갖는지 검증
  - 이미 생성된 약한 calendar JSP를 normalize가 다시 교체하는지 검증
